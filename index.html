Navigation Mode switching

j Move down M Middle line on screen : Enter command mode
k Move up L Bottom line on screen R Enter replace mode
l Move right     v Enter visual mode (highlighting)
10j Move down 10 lines     V Enter visual line mode (highlighting lines)
  esc Return to normal mode from insert or replace mode
gg First line of the file e The end of the current word esc+esc Return to normal mode from command or visual mode
G Last line of the file b Beginning of current word Ctrl+w Viewport (split) mode
:20 Line 20 of the file W Beginning of next word  
20G Line 20 of the file 20| Column 20 of the current line Copy/pasting
  Within vim
gt Go to next tab gT Go to previous tab y Yank
3gt Jump to the third tab hjkl Move cursor between viewports (splits) in viewport mode c "Change"; cut and enter insert mode
  C Change the rest of the current line
0 Beginning of current line d Delete; cut but remain in normal mode
^ First non-whitespace character of current line D Delete the rest of the current line after cursor
$ End of current line dd Delete the whole line
% Move to matching parenthesis, bracket or brace P Paste after the cursor
The left, right, up, and down arrow keys can also be used to navigate in some installations. P Paste before the cursor
x Delete characters after the cursor
Editing X Delete characters before the cursor
i Insert before current character Copy/paste commands operate on the specified range. If in visual mode, that range is the highlighted text. If in normal mode, that range is specified by a series of modifiers to the commands:
a Insert after current character
I Insert at the first non-whitespace character of the line cw Change one word
A Insert after the last non-whitespace character of the line c4w Change four words
o Insert a line below the current line, then enter insert mode c4l Change four letters
O Insert a line above the current line, then enter insert mode cc Change current line
r Overwrite one character and return to command mode 4x Change four characters after the cursor
U Undo 4p Paste five times after the cursor
Ctrl+R Redo  
. Repeat last insert mode action at the current location Modifiers work similarly for cut, delete, yank, and paste.
=G Reindent block  
gg=G Reindent the whole file From system clipboard
  :set paste Enter paste mode
Opening, closing, and saving files :set nopaste Exit paste mode
:w Save the current file Ctrl+Shift+V Paste into file, if in paste mode; Command+Shift+V for Mac
:wq Save the current file and close it; exits vim if no open files remain  
:w newname Save copy of current file as "newname", but continue editing the original file  
:sav newname Save copy of current file as "newname" and continue editing file "newname"    
:q! Close a file without saving    
:e somefile Opens file in the current buffer    
:x Write any changes to the file and close the file    
:vert Open any new window or file in a vertical split    
:term Open a terminal window in a new horizontal split    
Tabs Viewports/splits
:sp somefile Open a horizontal split window with a file :tabe somefile Open a file in a new tab
:vsp somefile Open a vertical split window with a file :tabc Close a tab
Viewport commands will only work in viewport (split) mode with at least two viewports open. Enter the viewport command keybind (Ctrl+w) between each of the below commands. :tabc index Close the tab at index, starting with 1
:tabo Close all other tabs except current
:tabs List all open tabs
= Make all viewports the same size :tabm 0 Move current tab to first index
- Decrease the current viewport by a single row or column, determined by whether the viewport is split vertically or horizontally :tabm Move current tab to last index
:tabm index Move current tab to index, starting with 0
+ Increase the current viewport by a single row or column, determined by whether the viewport is split vertically or horizontally :tabn Go to next tab
:tabp Go to previous tab
10- Decrease the current viewport by 10 rows or columns, determined by whether the viewport is split vertically or horizontally :tabfirst Go to first tab
:tablast Go to last tab
_ Increase a viewport to its maximum height  
| Increase a viewport to its maximum width Search
q Close the active viewport # Find the previous instance of the current word
r Rotate viewport to the right n Find the next instance of the word being searched for, in the direction specified by the last use of {*,#}
R Rotate viewport to the left
  N Find the previous instance of the word being searched for, in the direction specified by the last use of {*,#}
Replace
:s/foo/bar Replace the first occurrence of foo on the current line with bar /word Find the next occurrence of "word"
:[range]s/foo/bar/[flags] Replace foo with bar in range according to flags /word/c Find the next occurrence of "word", ignoring case ("\c" can appear anywhere in the sequence being searched for
 
Ranges /\<word\> Find the next occurrence of "word", where "word" is bounded by word boundaries (e.g. space, dash)
% The entire file
"<,"> The current selection; the default range while in visual mode :noh Un-highlight words

Regular expressions
Both vim's find and replace functions accept regular expressions. By default, vim assumes that some characters are part of a regular expression and these characters must be escaped with "\" to be searched for literally; these characters include (, ), *, ., ^, $. Other regular expression patterns are interpreted literally by default and must be escaped to be used as a part of a regular expression; these include +.
